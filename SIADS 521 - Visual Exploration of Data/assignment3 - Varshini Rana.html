Visualization Technique¶

The visualization technique I am planning to use is the Violin Plot.

A violin plot is a hybrid of a box plot and a kernel density plot. Like a box plot, it shows the median (the white dot at the center of the "violin"), the interquartile range (the black bar at the center of the "violin"), and the upper/lower adjacent values (the black lines stretched from the bar on either side). Unlike a box plot however, it depicts the entire distribution of numeric data using density curves (probability density function). The width of each curve corresponds with the frequency of data points in each region. Hence, unlike a box plot, a violin plot can help us visualize multimodal data (distributions with multiple peaks).

Violin plots are used when the distributions of numeric data need to be observed. They are especially useful when a comparison between distributions of multiple groups (i.e. the peaks, valleys, and tails of density curves) needs to be made. While overlapping density plots for different groups can be difficult to read and interpret, the violin plots for each group can just be placed side-by-side, making comparisons easy. Also unlike histograms which also show the distribution of data, the number of bins need not be specified for violin plots, which gives a more authentic shape of the distribution independent of the number of bins that would be chosen. Other visual elements such as those of Swarm Plot (the dots depicting individual observations), and box plot quartiles can also be overlayed on top of violin plots to provide more insights during comparisons if required.

It is important to consider that violin plots are useful to observe distributions of numeric data and not categorical data. The dataset should have continuous, numerical features. However, comparisons of numeric distributions across categories can be made. A violin plot also does not show sample size. Hence, if groups with different sample sizes are being compared, the sample sizes need to be explicitly made note of.

Visualization Library¶

The library I am going to use is Seaborn.

Seaborn is a Python package for creating statistical graphics using Python. It builds on top of matplotlib and integrates closely with pandas data structures. It is an open-source library created by Michael Waskom. It can be easily installed with the following command: pip install seaborn

The library readily displays visualizations in a Jupyter notebook or an IPython terminal with matplotlib mode enabled. Otherwise, the command matplotlib.pyplot.show() may need to be explicitly called.

I chose Seaborn for a number of reasons, the main one of them being that it creates the most aesthetically-pleasing visualizations of any visualization package I have used before. Moreover, its ease of use due to seamless integration with pandas indicates that its plotting functions operate on dataframes and arrays containing whole datasets, making the process of exploratory visualization of the data intuitive and convenient. The aesthetics are also freely and easily customizable. Moreover, Seaborn comes with various built-in statistical plots that just need a single line of code for implementation as opposed to matplotlib.

Seaborn is a declarative interface. Its dataset-oriented framework allows deeper focus on the interpretation of different elements of the plots rather than the nitty-gritty details of how to create them. However, this comes with the disadvantage of not being able to tinker with the finer nuts and bolts of the framework the way we can do with matplotlib. Some parts of the inner workings of the package may be a black box for the uninitiated, offering lesser control. Moreover, Seaborn has certain dependencies such as numpy, scipy, pandas and matplotlib, which means this library cannot be used independently.

Demonstration¶

I have procured the Titanic dataset from Kaggle (https://www.kaggle.com/c/titanic/data?select=train.csv). The dataset has been found to be sufficiently clean for further analysis. The main fields I am interested in are:

  1. Survived: 0 = passenger did not survive the Titanic disaster, 1 = passenger survived the Titanic disaster
  2. Class: 3 = third class passengers, 2 = second class passengers, 1 = first class passengers
  3. Gender: gender of passenger i.e. male or female
  4. Age: age of passenger

Using a violin plot, I want to see whether there is any difference in survival rates of men and women on comparing the distributions of the passengers based on their age, passenger class and fare.

The Violin Plot helps us determine the spread of the data and also gives us an idea of the median of the data (the white dots at the center of the bars). Setting the parameter dodge=True, we can visualize the violin plots of each group side-by-side for easier comparison. We can see here that passengers in the First Class seemed to be older, while those of the Third Class seemed to be younger in general among both genders.

Seaborn also gives us the functionality to overlay a Swarm Plot on top of this Violin Plot to visualize the individual samples better.

As an additional explorational aspect of the library for side-by-side comparisons of groups, Seaborn is able to readily create regression plots side-by-side as a facet. The function that enables us to do so is the lmplot, which is a hybrid of the regplot and the FacetGrid functionalities of Seaborn. It essentially denotes a scatter plot with a regression line.
